## Overview

This document defines the UI components required for each phase of Gastify's development. Components are organized by phase, with detailed specifications for layout, behavior, and animations.

**Design System Base:**

- Secondary: #3B82F6 (Blue)

- Background: #F9FAFB (Light gray)
- Text: #1F2937 (Dark gray)
- Muted: #6B7280 (Medium gray)

## Animation Library

### Standard Animations

```typescript
const animations = {
  // Entry animations
  fadeIn: {
    from: { opacity: 0 },
    to: { opacity: 1 },
    duration: 200,
    easing: 'ease-out'
  },
  slideUp: {
    from: { transform: 'translateY(20px)', opacity: 0 },
    to: { transform: 'translateY(0)', opacity: 1 },
    duration: 300,
    easing: 'ease-out'
  },
  slideInRight: {
    from: { transform: 'translateX(20px)', opacity: 0 },
    to: { transform: 'translateX(0)', opacity: 1 },
    duration: 250,
    easing: 'ease-out'
  },
  scaleIn: {
    from: { transform: 'scale(0.95)', opacity: 0 },
    to: { transform: 'scale(1)', opacity: 1 },
    duration: 200,
    easing: 'ease-out'
  },
  
  // Micro-interactions
  buttonPress: {
    to: { transform: 'scale(0.98)' },
    duration: 100,
    easing: 'ease-in-out'
  },
  checkmarkDraw: {
    // SVG path animation
    strokeDashoffset: { from: 100, to: 0 },
    duration: 300,
    easing: 'ease-out'
  },
  
  // Data visualizations
  progressBar: {
    from: { width: '0%' },
    to: { width: '{value}%' },
    duration: 500,
    easing: 'ease-out',
    stagger: 100 // for multiple bars
  },
  countUp: {
    duration: 800,
    easing: 'ease-out'
  },
  pieChart: {
    from: { strokeDashoffset: 100 },
    to: { strokeDashoffset: 0 },
    duration: 600,
    stagger: 100
  },
  
  // Celebrations
  confetti: {
    particles: 30,
    spread: 60,
    duration: 1500,
    colors: ['#10B981', '#3B82F6', '#F59E0B', '#EC4899']
  },
  bounce: {
    keyframes: [
      { transform: 'scale(1)' },
      { transform: 'scale(1.2)' },
      { transform: 'scale(0.9)' },
      { transform: 'scale(1.05)' },
      { transform: 'scale(1)' }
    ],
    duration: 500,
    easing: 'ease-in-out'
  }
};
```

### Animation Principles

1. **Speed:** Keep animations under 300ms for interactions, up to 800ms for data visualizations
2. **Easing:** Use ease-out for entries, ease-in for exits
3. **Purpose:** Every animation should reinforce meaning, never just decorate
4. **Performance:** Prefer transform/opacity over layout-affecting properties
5. **Reduce motion:** Respect prefers-reduced-motion media query

## Responsive Considerations

All components designed mobile-first for 375px width (iPhone SE) and scale up.

**Breakpoints:**
- Mobile: 375px - 640px
- Tablet: 641px - 1024px
- Desktop: 1025px+

**Key Adaptations:**
- Cards expand to fill width on mobile
- Side-by-side layouts on tablet/desktop
- Touch targets minimum 44x44px on mobile
- Navigation moves to sidebar on desktop
